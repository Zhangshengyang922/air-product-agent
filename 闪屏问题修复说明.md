# 登录闪屏问题修复说明

## 问题描述

用户反馈登录后出现闪屏现象,页面不断重定向。

## 问题原因

存在重定向循环:

1. 用户访问 `/` 主页
2. 后端检查请求头中的Token,发现没有
3. 后端返回登录页面 `login.html`
4. 登录页面检查localStorage中的Token,如果有就跳转到 `/`
5. 回到步骤1,形成无限循环

## 解决方案

### 修改内容

#### 1. 后端修改 (`src/main.py`)

**修改前**:
```python
@app.get("/")
async def root(request: Request):
    """主页,需要登录后才能访问"""
    token = verify_token_optional(request)
    
    if not token:
        # 未登录,返回登录页面
        login_file = static_dir / "login.html"
        if login_file.exists():
            return FileResponse(str(login_file))
        else:
            return {"message": "Please login", "status": "login_required"}
    
    # 已登录,返回主页
    index_file = static_dir / "index_elui.html"
    if index_file.exists():
        return FileResponse(str(index_file))
    else:
        return {"message": "Airline Product System", "status": "running"}
```

**修改后**:
```python
@app.get("/")
async def root(request: Request):
    """主页,直接返回主页HTML,由前端JavaScript检查登录状态"""
    index_file = static_dir / "index_elui.html"
    if index_file.exists():
        return FileResponse(str(index_file))
    else:
        return {"message": "Airline Product System", "status": "running"}
```

**说明**: 主页不再在服务端检查Token,直接返回主页HTML,由前端JavaScript检查并处理。

#### 2. 前端主页修改 (`static/index_elui.html`)

**添加立即执行检查**:
```javascript
// ==================== 立即检查登录状态(页面加载前) ====================
(function() {
    const token = localStorage.getItem('authToken');
    if (!token) {
        // 未登录,立即跳转到登录页
        window.location.href = '/login';
    }
})();
```

**说明**: 在页面加载的最开始立即检查Token,未登录则跳转,避免显示部分内容后闪烁。

**删除checkLoginStatus函数**:
```javascript
// 删除了之前的checkLoginStatus函数
// 因为已经在页面加载时检查过了
```

**修改initApp函数**:
```javascript
async function initApp() {
    // 移除了checkLoginStatus调用
    // ...
}
```

## 新的登录流程

### 正常登录流程

1. 用户访问 `http://localhost:8000/`
2. 主页加载时立即检查localStorage中的Token
3. 如果没有Token,立即跳转到 `/login`
4. 显示登录页面
5. 用户输入账号密码,点击登录
6. 调用 `/api/login` API
7. 登录成功,Token保存到localStorage
8. 跳转到主页 `/`
9. 主页加载时检查到Token,显示页面内容

### Token过期流程

1. 用户已登录,访问需要Token的API
2. API返回401状态码
3. 前端fetchAPI函数捕获到401
4. 清除localStorage中的Token
5. 跳转到登录页面

### 登出流程

1. 用户点击"登出"按钮
2. 调用 `/api/logout` API清除服务端Token
3. 清除localStorage中的所有登录信息
4. 跳转到登录页面

## 关键改进

### 1. 避免重定向循环
- 主页始终返回HTML,不根据Token返回不同内容
- Token检查完全在前端进行

### 2. 用户体验优化
- 页面加载时立即检查,避免显示部分内容后跳转
- 减少不必要的请求

### 3. 更好的控制
- 前端可以更灵活地处理登录状态
- 可以添加登录过期提醒等高级功能
