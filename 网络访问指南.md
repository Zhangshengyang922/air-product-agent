# 网络访问配置指南

## 🌐 访问模式说明

### 模式对比

| 模式 | localhost访问 | 局域网访问 | 使用场景 |
|------|--------------|-----------|---------|
| **仅本地** | ✅ | ❌ | 个人使用，不共享 |
| **本地+局域网** | ✅ | ✅ | 局域网内多人使用 |
| **仅局域网** | ❌ | ✅ | 局域网使用，禁止本机直接访问 |

---

## 📍 访问地址说明

### 1. 本地访问地址

```
http://localhost:8000
```
- 仅本机可访问
- 不受防火墙限制
- 离线时仍可用

```
http://127.0.0.1:8000
```
- 仅本机可访问
- 回环地址
- 离线时仍可用

### 2. 本机IP访问地址

```
http://192.168.1.100:8000  （替换为您的实际IP）
```
- 本机可访问
- 局域网可访问
- **网络断开后不可用**

### 3. 局域网访问地址

```
http://192.168.1.100:8000  （替换为您的实际IP）
```
- 局域网内所有设备可访问
- 需要在同一网络
- 需要防火墙放行8000端口

## 🔧 防火墙配置

### Windows防火墙放行8000端口

**方法1：添加入站规则**

1. 打开"高级安全Windows Defender防火墙"
   - Win+R → `wf.msc`

2. 新建入站规则
   - 左侧点击"入站规则"
   - 右侧"新建规则"

3. 配置规则
   - 规则类型：端口
   - 协议：TCP
   - 特定本地端口：8000
   - 操作：允许连接
   - 配置文件：域、专用、公用（全选）
   - 名称：产品管理系统8000

4. 完成

**方法2：命令行（需要管理员权限）**

```bash
# 添加防火墙规则
netsh advfirewall firewall add rule name="产品管理系统8000" dir=in action=allow protocol=TCP localport=8000

# 删除规则
netsh advfirewall firewall delete rule name="产品管理系统8000"

# 查看规则
netsh advfirewall firewall show rule name="产品管理系统8000"
```

**方法3：临时关闭防火墙（测试用）**

- 控制面板 → 系统和安全 → Windows Defender 防火墙
- 点击"启用或关闭Windows Defender防火墙"
- 关闭专用网络和公用网络的防火墙

⚠️ **注意**：关闭防火墙后记得重新开启！

## ⚠️ 网络断开后的使用

### 问题

当网络断开或电脑离线时：
- ✅ `localhost:8000` 仍可访问
- ✅ `127.0.0.1:8000` 仍可访问
- ❌ `http://192.168.1.100:8000` 不可访问

### 解决方案

**离线时使用：**
```
http://localhost:8000
```

**保持网络连接：**
- 确保有线/无线连接正常
- 路由器工作正常
- IP地址未改变

### 固定IP地址（推荐）

为避免IP变化，设置固定IP：

1. 打开"网络和共享中心"
2. 更改适配器设置
3. 右键网卡 → 属性
4. Internet 协议版本4 (TCP/IPv4) → 属性
5. 使用下面的IP地址：
   - IP地址：`192.168.1.100`（选择未占用的IP）
   - 子网掩码：`255.255.255.0`
   - 默认网关：`192.168.1.1`（路由器IP）
   - DNS服务器：`114.114.114.114`

## 🔍 故障排查

### 问题1：局域网无法访问

**检查清单：**

- [ ] 防火墙是否放行8000端口
- [ ] IP地址是否正确
- [ ] 是否在同一局域网
- [ ] 服务是否正常运行
- [ ] 网络连接是否正常

**排查步骤：**

1. **检查防火墙**
   ```bash
   netsh advfirewall show allprofiles
   ```

2. **检查端口**
   ```bash
   netstat -ano | findstr :8000
   ```

3. **测试连接**
   ```bash
   ping 192.168.1.100
   ```

4. **查看服务状态**
   ```bash
   # 在局域网设备上访问
   http://192.168.1.100:8000
   ```

### 问题2：IP地址变化

**解决方法：**

1. 设置固定IP（推荐）
2. 使用DNS服务
3. 每次启动时查看新IP

### 问题3：服务无法启动

**检查：**

1. **端口被占用**
   ```bash
   netstat -ano | findstr :8000
   taskkill /F /PID <进程ID>
   ```

2. **Python环境**
   ```bash
   python --version
   ```

3. **依赖包**
   ```bash
   pip install fastapi uvicorn
   ```
